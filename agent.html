🧠 Agent

智能体架构与自主决策

什么是 Agent

Agent（智能体）是基于 LLM 构建的自主决策系统。与单次 LLM 调用或固定 Chain 不同，Agent 能根据目标动态规划步骤、调用工具、观察结果，并循环迭代直到完成。

核心特征：LLM 充当推理引擎（Reasoning Engine），而非单纯的文本生成器。

核心组件

  • 感知（Perception）：解析用户输入与当前环境状态
  • 规划（Planning）：把目标拆解为可执行步骤序列
  • 记忆（Memory）：短期上下文 + 长期知识库（向量存储）
  • 工具使用（Tool Use）：调用外部能力（搜索、代码、API）
  • 行动（Action）：执行步骤并把结果回写到记忆

典型模式：ReAct

ReAct（Reason + Act）是最经典的 Agent 循环范式：

while not task_done:
    thought  = llm.think(context, history)
    action   = llm.select_tool(thought)
    result   = execute(action)
    context  = context + result

每个循环包含 4 步：思考 → 选择工具 → 执行 → 观察。LLM 决定何时终止循环、是否需要纠正方向。

实战注意事项

  1. 工具描述决定上限：工具的 name、description、parameters schema 是 LLM 决策的唯一依据，描述不清则 Agent 调用混乱
  2. 上下文是稀缺资源：每次循环都累积 token，需要设计记忆压缩/总结机制
  3. 必须设 max_steps：避免无限循环或成本爆炸
  4. 错误恢复：工具失败时给 LLM 重试或换路径的机会，而不是直接终止
