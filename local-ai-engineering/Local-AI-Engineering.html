🖥️ 专题说明

这一专题聚焦"单机场景"：不追求大规模服务化的极致吞吐，追求在一台 Mac / 一张显卡 / 一台笔记本上，把模型真的跑起来，并且针对具体任务微调到能用。内容分两个集群：

  • 部署　硬件选型 → 模型格式与量化 → 本地推理引擎 → 服务化部署，覆盖 Mac（统一内存 / Metal）、Linux（独立显存 / CUDA·ROCm，事实上的一等公民）、Windows（WSL2 绕路居多）三条不同起点
  • 训练与微调　方法选型 → 数据集构造 → 工具链对比 → 四类实战场景（代码助手 / 领域问答 / 结构化抽取 / 工具调用）→ 效果评估 → 合并导出，形成"部署→微调→再部署"的完整闭环

一个需要先说清楚的边界：AI系统工程专题（暂未开始）讲的是服务端/生产级推理引擎的机制原理——vLLM / SGLang 内部怎么做 continuous batching、PagedAttention 怎么省显存，面向大规模服务化场景。这里的 Local AI Engineering 讲的是单机/消费级硬件上的部署与微调实操——两者会共用一些底层概念（比如量化），但本专题只从"这台机器该选哪种格式、能不能跑得动、怎么针对我的任务微调"的实操角度切入，不重复讲算法内部原理。

📚 集群一：部署

🧰

硬件与环境选型

同样是"能跑本地模型"，Mac 的统一内存、Linux 的独立显存、Windows 的 WSL2 绕路，起点差在哪？一篇讲清三大平台的算力和生态前提，决定后面所有选择的上限。
🗜️

模型格式与量化实践

同一个模型，GGUF、MLX、GPTQ/AWQ/EXL2 该选哪个？一篇讲清格式和量化方案在你这台机器上到底跑不跑得动，而不是纠结算法内部怎么算误差补偿。
⚙️

本地推理引擎全景

llama.cpp、Ollama、LM Studio、MLX-LM、vLLM 本地模式，谁在哪个平台是"一等公民"、谁是"能用但要自己填坑"？一篇讲清怎么选。
📡

本地服务化部署

模型跑起来之后，怎么包成一个常驻 API 让其他程序调用？一篇讲清三大平台的容器化、进程管理和开机自启差异。

📚 集群二：训练与微调

🧩

微调方法选型

全参数、LoRA、QLoRA、Adapter，本地资源约束下哪些理论可行、哪些算力门槛直接劝退你？一篇讲清怎么选。
🗂️

任务专属数据集构造实战

"我想让模型做 X"要怎么翻译成可训练样本？指令对、领域 QA、function-calling 格式、分类/抽取标注，每种任务的数据形态都不一样。
🛠️

微调工具链三平台对比

peft、unsloth、axolotl、LlamaFactory、MLX-LM 微调，同一套代码在 Mac 上跑不跑得动，Windows 是不是必须切 WSL2？
💻

实战一：代码助手微调

数据量级、方法选择、平台可行性三件套，针对"让模型更懂我的代码风格/框架"这个具体任务逐一讲透。
💬

实战二：领域知识/客服问答微调

把一堆领域文档和历史问答，变成能让模型"说话像内部专家"的训练数据，本地能不能练出来？
🏷️

实战三：结构化抽取与分类微调

让模型稳定输出固定格式的抽取/分类结果，比"聊得像"更看重精度和一致性，数据和评估方式都要重新设计。
🔧

实战四：工具调用能力微调

让本地小模型学会稳定地按格式发起工具调用，是四类场景里最容易"看起来学会了、实际一碰就崩"的一类。
🔍

微调效果评估：没有云端评测集群时怎么办

本地怎么构造小规模测试集？怎么分辨"真学会了新任务"还是"过拟合到了训练集的语气"？常见的自我欺骗式评估陷阱。
🔄

LoRA 合并与重新导出部署

微调完的 adapter 怎么合并回底座模型，再转回 GGUF/MLX 重新部署？闭环回到集群一，"部署→微调→再部署"就此循环闭合。
