🌱 Pretraining：模型的「三观」，是怎么被灌进去的？

为什么模型会一本正经编造事实、知识永远停在某个截止日期？一篇讲清预训练到底在做什么 · 双栖科普

零、引言：模型的「世界观」，是被谁定下来的？

先扔几个问题：

  • 为什么模型会一本正经地编造一个根本不存在的引用、论文、法条，讲得跟真的一样？
  • 为什么问模型「现在的 XX 是谁」，它给出的答案常常是几个月甚至几年前的信息？
  • 为什么同一个底座模型，经过不同方式训练之后，说话风格能判若两人？
  • 为什么训练一次模型要花几千万美元、几个月时间，不能像软件更新一样「打个补丁」就完事？
  • 为什么有时候模型会一字不差地「背」出一段训练数据里出现过的原文？

这五个现象，根子都在同一个阶段：Pretraining（预训练）。上一篇讲的 Attention / Transformer 只是搭好了骨架结构，模型此时什么都不「知道」；预训练，就是往这个骨架里灌入海量文本，把语言能力和世界知识压进参数里的过程。

💡 Attention 给了模型一副能理解上下文的大脑结构。预训练，是往这副大脑里塞进人类写下的几乎所有文字，让它自己总结出规律。

一、Pretraining 在 AI 世界里的坐标

紧接 Attention / Transformer 管线往下走：

Tokenizer → Token ID → Embedding → Attention / Transformer（骨架结构）
    ↓
Pretraining（往骨架里灌数据，学出语言和知识）  ← 本篇
    ↓
Alignment（SFT / RLHF / DPO，教模型听懂指令、说真话）
    ↓
Inference（逐 Token 预测）
    ↓
LLM 输出

Transformer 决定的是「形状」，预训练决定的是「里面装了什么」。同样的架构，用不同的数据、不同的规模训练，出来的模型能力天差地别。

这一阶段的影响会渗透到整个工程体系里：

工程概念 跟 Pretraining 的关系
Context Engineering你会遇到的「知识截止日期」问题，根源就在预训练语料的采集时间范围
RAG EngineeringRAG 之所以必要，正是因为预训练数据是一次性的静态快照，模型天生不知道训练之后发生的事
Prompt Engineering某些提问方式效果特别好，往往是因为这类表达在预训练语料里出现的频率高、模式熟悉
Hallucination（后续专篇）幻觉现象的根源之一，就藏在预训练阶段的训练目标本身，这篇会先埋下伏笔
💡 模型不是「学会了世界」，是「记住了训练数据里关于世界的文字描述」——这两者之间的落差，是后面几乎所有「翻车现场」的起点。

二、为什么不能直接靠人工标注数据训练？

在预训练这套方法流行之前，让 AI「学会语言」的思路大致有两种：

方案 怎么做 致命问题
人工规则系统（专家系统） 语言学家手写语法规则和知识库 人类语言的灵活性和歧义远超规则能覆盖的范围，遇到没预料到的表达就直接失灵
全监督学习 人工给每条数据标注「正确答案」 标注成本随数据量线性甚至更快增长，覆盖不了互联网规模的文本；而且很多语言任务没有唯一「正确答案」

预训练走的是第三条路：自监督学习（Self-Supervised Learning）。核心任务很朴素——给模型看一段文字的前半部分，让它预测下一个词是什么，然后拿真实的下一个词去对答案，算出误差，反向调整参数。这个过程完全不需要人工标注，因为文本本身自带「标准答案」——下一个词是什么，语料里现成写着。这也是为什么互联网上几乎无限量的原始文本，第一次变成了可以直接拿来训练的「标注数据」。

💡 预训练最聪明的地方，不是算法多复杂，而是把「预测下一个词」这个任务设计成不需要人工标注——文本自己就是自己的老师。

三、Pretraining 到底是怎么工作的？

核心任务：给定前面的文字，预测下一个 Token 是什么。模型算出一个「下一个词是每个候选 Token 的概率分布」，跟真实答案对比算出误差，通过反向传播微调参数，让下次遇到类似上下文时，猜得更准一点。这个过程在万亿级别的 Token 上重复无数遍，模型的参数里就逐渐沉淀出语法规律、常识、专业知识——不是被人为「存」进去的，是被「猜下一个词」这个单一任务反复训练出来的副产品。

这里有一个研究领域专门在讨论「规模到底怎么影响效果」——通常称为 Scaling Laws：数据量、参数量、训练算力这三者共同决定模型能力的上限，三者之间存在大致可预测的规律，单独堆某一项而其他两项跟不上，效果会打折扣。

还有一个更反直觉的现象，叫涌现能力（Emergent Abilities）：某些能力（比如多步推理）在小规模模型上几乎完全不存在，但当规模跨过某个临界点后会突然出现，不是随着规模平滑地线性增长——这也是为什么「参数量到底要多大」在业内一直是个持续在被重新验证的问题。

💡 预训练不是「教模型知识点」，是「让模型在无数次猜下一个词的过程中，自己总结出知识点长什么样」——这也是为什么模型偶尔会猜错、猜得很自信，却是错的。

四、Pretraining 会如何影响模型能力？

只讲真实案例。

为什么模型会一本正经编造不存在的事实？

翻车现场：让模型列出某个领域的参考文献，它给出的书名、作者、期刊看起来完全合理，一查却根本不存在。

根因：预训练的训练目标是「预测最可能通顺的下一个词」，不是「陈述被核实过的事实」。语言的流畅性和内容的真实性，在这个训练目标里从来不是同一件事——模型学会了「参考文献大概长什么样子」这种文体规律，但没有一个内置的事实核查机制去验证它编出来的内容是否真实存在。

为什么模型的知识永远停在某个时间点？

翻车现场：问模型某个近期事件、某个岗位现在是谁在任，得到的答案是几个月甚至几年前的过时信息。

根因：预训练语料是在某个时间节点之前收集的静态快照，训练完成后，模型的参数就固定了——之后发生的事，从没出现在训练数据里，模型自然无从知晓。这也是为什么 RAG、联网搜索这类「外部信息接入」手段，是应用层的必需品而不是锦上添花的可选项。

为什么模型有时候会一字不差地「背」出训练数据原文？

翻车现场：针对某些高频出现的内容（比如常见名言、经典代码片段），模型的输出和某段真实存在的训练语料几乎逐字重合。

根因：对于在训练数据里反复出现次数很多的片段，模型有时候并不是「理解后重新组织语言」，而是直接把这段文字的具体序列「记」进了参数里。这也是训练数据的版权和隐私问题，会从学术讨论变成现实工程和法律议题的原因之一。

💡 幻觉不是模型「学坏了」，是训练目标从一开始就没有把「说真话」设成唯一标准——预测通顺，和陈述真实，是两件被放在一起但本质不同的事。

五、为什么开发者一定要懂 Pretraining？

Context Engineering：知识截止日期问题的根源，就在预训练语料的时间范围——理解了这一点，才知道为什么「喂给模型最新信息」这件事永远绕不开。

RAG Engineering：RAG 存在的根本原因，正是预训练数据天生是静态、有截止时间的快照——RAG 不是锦上添花的优化，是弥补预训练固有局限的必需手段。

幻觉问题的认知校正：幻觉不是可以靠「多训练一点数据」就根除的 bug，它是训练目标本身带来的固有特性——后续的 Alignment 阶段能缓解、能让模型学会「不确定就说不知道」，但没法从根子上消除。

往下一篇的伏笔：预训练让模型学会了「语言」和「知识」，但此时的模型还不会好好听指令、不知道该拒绝什么、也不擅长以对话的方式回应——这是下一篇 Alignment（SFT / RLHF / DPO）要解决的问题。

六、常见误区

误区一：幻觉是训练数据不够多导致的，数据越多幻觉越少

真相：幻觉根源在训练目标本身（预测流畅性，不是事实核查），单纯堆数据量不能从根本上解决，需要额外的对齐和校验机制配合。

误区二：模型能在后台「联网」实时更新自己的知识

真相：预训练是一次性的静态训练过程，参数不会在部署后自动更新；所谓「联网」能力，本质是调用外部工具（搜索、RAG），不是模型自身知识在更新。

误区三：参数量越大，模型一定越强

真相：Scaling Laws 说明数据量、参数量、算力三者需要匹配才能发挥效果，单独堆参数而数据或算力没跟上，效果会打折扣——训练充分的小模型完全可能超过训练不充分的大模型。

误区四：模型训练一次，之后就不用再训练了

真相：语言习惯、热点话题、世界知识都在变化，模型需要定期用新数据重新训练或做增量更新，才能不被时代甩下。

💡 理解预训练，本质是理解模型能力的「天花板」从哪来——后面所有的对齐、微调、提示工程，都是在这个天花板之下做优化，不能凭空突破预训练没学到的东西。

七、总结：模型的常识，是被「猜」出来的

给小白的话：模型的知识不是被人一条条「教」进去的，而是在海量文本上反复「猜下一个词」猜出来的——理解了这一点，就不会再惊讶它偶尔一本正经地编故事，也不会再指望它知道训练截止之后发生的事。

给开发者的话：Pretraining 决定了模型能力的天花板和固有局限——知识的时效性、幻觉的根源、参数规模和效果的关系，都是在这一阶段被定下来的。这也是为什么应用层的 Context Engineering、RAG Engineering 不是「锦上添花的优化技巧」，而是在弥补预训练阶段就注定存在的结构性短板。搞懂这一层，才能对「模型能做到什么、做不到什么」有一个不靠猜的判断标准。

💡 Attention 给了模型理解上下文的能力，预训练给了它关于世界的语言化记忆——但记忆终究停在某个时间点，之后的世界，需要你自己喂给它看。
