🪙 Tokenization：为什么 AI 根本没有看过文字？

为什么中文更贵、XML 更好、AI 数不清字母？一篇讲清 Token 的世界 · 双栖科普

跳转：引言 · 坐标 · 为什么不直接读文字 · 怎么工作的 · 怎么影响能力 · 为什么开发者要懂 · 常见误区 · 总结

零、引言："Token 不就是字数吗？"

先扔几个问题，看你中过几个：

  • 为什么 API 按 Token 收费，而不是按字数？
  • 为什么 Context Window 用 Token 做单位，而不是字数？
  • 为什么你问 GPT"strawberry 里有几个 r"，它能给出五个不同的答案？
  • 为什么 XML 标签在 Prompt 里比特殊符号更稳定？
  • 为什么一段 Base64 能瞬间吃掉几万 Token？

答案只有一句话：AI 根本没有看到文字。它看到的全部东西，叫 Token。

💡 你以为模型在读你的话。实际上，它连一个汉字都没见过。

一、Tokenization 在 AI 世界里的坐标

Human Language（你输入的文字）
    ↓
Tokenizer（把文字切成 Token）
    ↓
Token（一串数字 ID）
    ↓
Embedding（把数字 ID 映射成向量）
    ↓
Transformer（在向量空间里做注意力运算）
    ↓
Inference（逐 Token 预测下一个 Token）
    ↓
LLM（把 Token ID 序列映射回文字，返回给你）

Tokenization 卡在人类语言和数学运算之间——它是文字进入 AI 世界的第一道门。

工程概念跟 Token 的关系
Prompt Engineering你写的 Prompt，模型看到的是一串 Token ID，不是汉字
Context EngineeringContext Window 的容量单位是 Token，不是字符数
RAGChunk 按 Token 数切，不是按页数切
AgentMemory 压缩的本质是减少 Token 数
API Cost计费单位是 Token，成本优化就是 Token 优化

二、为什么 AI 没有直接阅读文字？

方案怎么切致命问题
按字符一个字一个 ID序列太长，Attention O(n²) 直接炸
按单词一个词一个 ID词表爆炸，生词变 <UNK>
按句子一句话一个 ID完全没有泛化能力

NLP 最终收敛到了折中方案：Subword（子词）——把高频出现的"字母组合"当作最小单元。

三、Tokenizer 到底是怎么工作的？

BPE 核心逻辑：反复合并最高频的相邻对。没有语义理解，没有语法规则，纯粹是统计。

💡 Tokenizer 学到的不是语义，只是高频组合。

四、Token 会如何影响模型能力？

为什么 GPT 数不清 "strawberry" 里有几个 r？

Tokenizer 切出来的是类似 str-aw-berry 这样的片段——每个 Token 内部的具体字母，模型根本看不到。

为什么中文更贵？

大多数主流 Tokenizer 的训练语料以英文为主。同样信息量，中文需要的 Token 数更多。

为什么 Base64 能瞬间吃掉几万 Token？

Base64 字符分布完全随机，Tokenizer 退化到几乎逐字符切分。

五、为什么开发者一定要懂 Token？

Prompt Engineering：为什么 XML / JSON / Markdown 更稳定？因为它们在训练语料里大量出现，通常被切成完整 Token。

Context Engineering：窗口统计的是 Token，不是字符。不同语言的 Token/字符比完全不同。

RAG：Chunk 按 Token 切，不是按页切。

Agent：Memory 压缩的目标就是把 Token 数压回安全范围。

六、常见误区

  • 误区一：Token ≈ 字数——1 个中文字符 ≈ 1~2 个 Token
  • 误区二：Token ≈ 单词——高频单词如 "the" 是一个 Token，"university" 可能是两个
  • 误区三：减少字符就能减少 Token——缩写可能恰好跟你省掉的高频组合一样大
  • 误区四：不同模型的 Tokenizer 差不多——每个模型的 Tokenizer 是独立训练的

七、总结：语言属于人类，Token 属于模型

给小白的话：下次跟 AI 聊天成本高了、回答跑偏了，可能不是你表达不清，是你的文字在被模型接收的第一步就被"肢解"成了你完全没预料到的样子。

给开发者的话：Tokenization 是在你写的每一行 Prompt、做的每一次上下文裁剪、算的每一笔 API 成本里反复出现的基础变量。

💡 人类认为语言由字、词和句子组成。模型认为语言由 Token 组成。
