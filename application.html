📦 专题说明

这一专题聚焦"怎么把 LLM 用起来"——从"写一句 Prompt"到"持续自主运行的调度系统"，按"从内到外、从单次到持续"的顺序讲清八块拼图：

  • Prompt Engineering　是单轮输入的指令写法，决定"怎么说清楚任务"
  • Context Engineering　是单次会话的信息组织，决定"喂什么、不喂什么"
  • RAG Engineering　是外部知识的供应链，决定"实时信息从哪来"
  • Skill　是常用能力的封装，决定"哪些动作可以一键调用"
  • Agent　是执行载体，决定"谁来做、做到什么程度"
  • Harness Engineering　是给 Agent 画的红线，决定"在多大范围内自主"
  • Workflow Engineering　是多步协作的编排，决定"多步怎么串成一条链"
  • Loop Engineering　是持续运行的调度系统，决定"什么时候该停、什么时候该交人"

八篇互相咬合，建议按顺序读——前一篇文章里的概念，会在下一篇里被重新调用。
