📦 专题说明

这一专题聚焦"怎么把 LLM 用起来"——围绕 Agent 这个执行载体，依次讲清楚它的六块拼图：

  • Agent　是执行的躯壳，决定"谁来做"
  • Skill　是装进躯壳的专业知识，决定"做得多专业"
  • Harness Engineering　是给它画的红线，决定"在多大范围内自主"
  • Context Engineering　是它的工作台，决定"一次能拿多少信息"
  • Prompt Engineering　是它的指令，决定"怎么理解任务"
  • Loop Engineering　是它的迭代闭环，决定"怎么越做越好"

六篇互相咬合，建议按顺序读——前一篇文章里的概念，会在下一篇里被重新调用。
